## 2. АНАЛИЗ КОНФИГУРАЦИИ СЕРВЕРА

### 2.1 Текущая команда запуска

```bash
ExecStart=/usr/bin/npx tsx server/index.ts
```

### 2.2 Выявленные проблемы

- `tsx` — медленный запуск (компиляция TypeScript при каждом запуске)
- Нет production build
- Нет PM2/cluster для многоядерности

### 2.3 Рекомендуемая команда запуска

```bash

npm run build:server  # Сборка в dist-server/
node --env-file=.env dist-server/index.js

# Или использовать скомпилированную версию:
node --env-file=.env dist-server/index.js
```

## 4. ЗАЩИТА PHPMYADMIN

### 4.1 Текущая реализация

```typescript
// server/index.ts
app.get('/phpmyadmin/*', async (c, next) => {
  // Проверка сессии и роли ADMIN
  // Проблема: phpmyadmin на economikus.ru/phpmyadmin — это НЕ маршрут Hono!
})
```

### 4.2 Проблема

phpmyadmin находится на https://economikus.ru/phpmyadmin — это НЕ маршрут Hono!

Это:
- Либо отдельный сервис/контейнер
- Либо настроенный в nginx/apache как прокси

**Защита на уровне Hono НЕ РАБОТАЕТ** для внешнего сервиса.

### 4.3 Решение

Настроить защиту на уровне **nginx**:

```nginx
# /etc/nginx/sites-available/economikus
location /phpmyadmin {
    allow 127.0.0.1;
    allow YOUR_IP;
    deny all;
    
    auth_basic "Admin Only";
    auth_basic_user_file /etc/nginx/.htpasswd;
    
    proxy_pass http://localhost:8080;
}
```

## 6. АНАЛИЗ РИСКОВ И ПОСЛЕДСТВИЙ

### 6.1 Code Splitting для @mantine, recharts, react-markdown

| Риск | Вероятность | Последствия | Митигация |
|------|-------------|-------------|-----------|
| **Нарушение tree-shaking** — @mantine/core имеет много взаимозависимостей | Средняя | Увеличение размера вместо уменьшения | Использовать `@mantine/core` без отдельного tree-shaking, проверить результат билда |
| **Ошибки при динамической загрузке** — компонент вызывается до загрузки | Средняя | "Component is not mounted" / белый экран | Добавить Suspense и ErrorBoundary |
| **Стили не применяются** — CSS чанки не загружаются | Низкая | Без стилей | Mantine автоматически подключает styles |
| **Recharts ломается при lazy loading** — графики рендерятся до загрузки | Высокая | Ошибки рендеринга | Графики должны рендериться на клиенте, обернуть в Suspense |

### 6.2 Gzip/Brotli сжатие

| Риск | Вероятность | Последствия | Митигация |
|------|-------------|-------------|-----------|
| **CPU нагрузка на сервере** — сжатие требует ресурсов | Низкая | Задержки при высокой нагрузке | brotli более эффективен, но медленнее; использовать для статики |
| **Проблемы с бинарными файлами** — уже сжатые файлы (images, zip) | Низкая | Незначительное увеличение размера | Настроить исключения для изображений |
| **Несовместимость со старыми браузерами** | Низкая | Контент не отображается | Настроить fallback на gzip |

### 6.3 Динамические импорты (React.lazy)

| Риск | Вероятность | Последствия | Митигация |
|------|-------------|-------------|-----------|
| **FOUC (Flash of Unstyled Content)** — мигание при загрузке | Средняя | Плохой UX | Использовать Suspense с хорошим fallback |
| **Ошибка загрузки чанка** — медленный интернет, обрыв связи | Средняя | "Failed to load chunk" | Добавить ErrorBoundary с retry |
| **Навигация до загрузки** — пользователь уходит со страницы | Низкая | memory leak | Отмена запросов при unmount |
| **Нарушение типов** — TypeScript не видит динамические импорты | Низкая | Ошибки компиляции | Использовать `typeof` или `Module` |

### 6.4 Перенос аватаров из base64 в файловую систему

⚠️ **НАИБОЛЕЕ РИСКОВАННАЯ ОПЕРАЦИЯ**

| Риск | Вероятность | Последствия | Митигация |
|------|-------------|-------------|-----------|
| **Потеря данных** — старые аватары недоступны | Высокая | Пользователи теряют аватарки | Сделать МИГРАЦИЮ, не удалять старое поле |
| **Нарушение ссылок** — где-то ещё используется base64 | Средняя | Ошибки отображения | Найти все использования avatarUrl |
| **Права доступа** — папка не доступна для записи | Средняя | Ошибки загрузки | Создать папку с правильными правами |
| **Место на диске** — аватары занимают место | Низкая | Заполнение диска | Добавить cleanup задачу |

### 6.5 Кэширование (Cache-Control)

| Риск | Вероятность | Последствия | Митигация |
|------|-------------|-------------|-----------|
| **Устаревшие данные** — пользователь видит старое | Средняя | Баги отображения | Использовать versioned URLs |
| **Приватные данные в кэше** — утечка данных | Высокая | Утечка чувствительных данных | `private` для авторизованных |
| **Медиафайлы не обновляются** — новый аватар не виден | Средняя | Старый аватар | Добавить ?v=timestamp к URL |

### 6.6 Изменение запуска сервера

| Риск | Вероятность | Последствия | Митигация |
|------|-------------|-------------|-----------|
| **dist-server устарел** — не включает последние изменения | Средняя | Старый функционал | Автоматизировать билд при деплое |
| **Ошибки при билде** — TypeScript ошибки | Низкая | Не запускается | Проверить `npm run build:server` локально |
| **Переменные окружения** — не подхватываются | Средняя | Ошибки подключения к БД | Проверить `.env` файл |
