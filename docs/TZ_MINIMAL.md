# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## на разработку веб-приложения «Экономикус»

---

## 1. Наименование и область применения

### Наименование

Веб-приложение «Экономикус» (далее — Система).

### Область применения

- Предоставление доступа к образовательным курсам по финансовой грамотности, инвестициям и управлению личными финансами;
- Создание и публикация образовательного контента авторами (курсы, уроки, тесты, калькуляторы);
- Управление процессом обучения (отслеживание прогресса, история просмотров, сертификаты);
- Монетизация контента через систему подписок;
- Привлечение целевой аудитории (начинающие инвесторы, предприниматели, все интересующиеся финансовой грамотностью).

---

## 2. Основание для разработки

Разработка ведётся в рамках учебного/коммерческого проекта, а также в связи с необходимостью:

- Создания современной образовательной платформы для изучения финансов;
- Автоматизации процесса обучения и отслеживания прогресса;
- Обеспечения удобного интерфейса для авторов по созданию контента;
- Обеспечения администраторам инструментов управления платформой;
- Монетизации образовательного контента через подписки;
- Продвижения продукта в поисковых системах (SEO).

---

## 3. Назначение разработки

### Для пользователей (обучающихся):

- Просмотр каталога курсов с фильтрацией и поиском;
- Прохождение курсов и уроков различных типов (текст, видео, аудио, тесты, калькуляторы);
- Отслеживание прогресса обучения;
- Получение сертификатов о прохождении курсов;
- Добавление материалов в избранное;
- Просмотр истории просмотров;
- Оформление подписки на премиум-контент;
- Взаимодействие через комментарии и реакции.

### Для авторов:

- Создание и редактирование курсов;
- Управление модулями курсов (drag&drop);
- Создание уроков различных типов контента;
- Отправка контента на модерацию;
- Просмотр аналитики по своим курсам.

### Для администраторов:

- Управление пользователями (блокировка, изменение ролей);
- Модерация контента (публикация, архивирование);
- Управление тегами;
- Рассмотрение заявок на статус автора;
- Просмотр статистики платформы.

### Для поисковых систем:

- Индексация страниц через sitemap.xml;
- Семантическая структура заголовков (H1-H6);
- Мета-теги для каждой страницы;
- Файл robots.txt.

## 6. Требования к надёжности

- Все данные, передаваемые через формы, должны проходить валидацию на стороне клиента (React) и сервера (Zod);
- Защита от CSRF-атак через SameSite=Strict cookie;
- Защита от SQL-инъекций обеспечивается использованием Prisma ORM;
- Все пользовательские данные проходят санитизацию;
- Пароли хранятся в базе данных в захешированном виде (bcryptjs, 12 раундов);
- При некорректном вводе данных формы пользователь получает сообщение об ошибке;
- Система должна корректно обрабатывать отсутствие соединения с базой данных;
- Глобальный обработчик ошибок (ErrorBoundary) на клиенте;
- Логирование ошибок на сервере.

## 12. Программное обеспечение для работы веб-сервера

Для запуска и тестирования веб-приложения используется среда разработки на базе Node.js с инструментами сборки Vite.

**Node.js** – среда выполнения JavaScript-кода вне браузера. Используется для запуска серверной части приложения и инструментов сборки. Рекомендуемая версия: Node.js v18.0.0 или выше (LTS).

**Vite** – современный инструмент сборки и разработки фронтенд-приложений. Обеспечивает быструю перезагрузку модулей (HMR), оптимизацию Production-сборки и проксирование API-запросов.

**React** – JavaScript-библиотека для создания пользовательских интерфейсов. Версия: React 19.1.0.

**Hono** – легковесный веб-фреймворк для Node.js, используемый для серверной части API. Версия: Hono 4.12.7.

**MySQL** – СУБД (система управления базами данных), используемая для хранения информации о пользователях, курсах, уроках, подписках и других данных. Рекомендуемая версия: MySQL 8.0 или выше.

**Prisma** – ORM (Object-Relational Mapping) для работы с базой данных MySQL. Обеспечивает типобезопасные запросы, миграции и генерацию схемы базы данных.

**nginx** – веб-сервер и обратный прокси-сервер. Используется для обслуживания статических файлов (HTML, CSS, JS, изображения), проксирования API-запросов к Node.js-приложению, настройки SSL/TLS-шифрования и кэширования.

**systemd** – система инициализации и управления службами в Linux. Используется для автоматического запуска Node.js-приложения при старте сервера, перезапуска при сбоях и управления логами.

**npm** – пакетный менеджер для Node.js. Используется для установки зависимостей и запуска скриптов проекта.

### Алгоритм настройки и запуска проекта

#### Локальная разработка

1. **Установить Node.js**
   - Скачать с официального сайта https://nodejs.org/ (версия LTS).
   - Установить, следуя инструкциям установщика.
   - Проверить установку командой: `node -v` и `npm -v`.

2. **Установить MySQL**
   - Скачать с официального сайта https://dev.mysql.com/downloads/mysql/.
   - Установить, задать пароль для пользователя `root`.
   - Создать базу данных через MySQL Workbench или командную строку:
   ```sql
   CREATE DATABASE economikus CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. **Клонировать репозиторий проекта**
   ```bash
   git clone <url-репозитория>
   cd economikus
   ```

4. **Установить зависимости проекта**
   ```bash
   npm install
   ```

5. **Настроить переменные окружения**
   - Создать файл `.env` в корне проекта.
   - Указать параметры подключения к БД:
   ```
   DATABASE_URL="mysql://root:password@localhost:3306/economikus"
   ```

6. **Применить миграции базы данных**
   ```bash
   npx prisma migrate dev
   ```

7. **Запустить сервер разработки**
   ```bash
   npm run dev
   ```
   - Приложение будет доступно по адресу http://localhost:5173
   - API сервер запускается на http://localhost:3000

#### Продакшн-сервер (Linux)

8. **Установить и настроить nginx**
   - Установить nginx:
   ```bash
   sudo apt update
   sudo apt install nginx
   ```
   - Создать конфигурацию сайта `/etc/nginx/sites-available/economikus`:
   ```nginx
   server {
       listen 80;
       server_name economikus.ru www.economikus.ru;

       location / {
           root /var/www/economikus/dist;
           try_files $uri $uri/ /index.html;
       }

       location /api {
           proxy_pass http://127.0.0.1:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```
   - Активировать конфигурацию:
   ```bash
   sudo ln -s /etc/nginx/sites-available/economikus /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

9. **Настроить systemd-сервис**
   - Создать файл сервиса `/etc/systemd/system/economikus.service`:
   ```ini
   [Unit]
   Description=Economikus API Server
   After=network.target mysql.service

   [Service]
   Type=simple
   User=www-data
   WorkingDirectory=/var/www/economikus
   ExecStart=/usr/bin/node dist-server/index.js
   Restart=on-failure
   RestartSec=10
   StandardOutput=syslog
   StandardError=syslog
   SyslogIdentifier=economikus
   Environment=NODE_ENV=production

   [Install]
   WantedBy=multi-user.target
   ```
   - Активировать сервис:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable economikus
   sudo systemctl start economikus
   ```

10. **Сборка и деплой**
    ```bash
    npm run build
    sudo cp -r dist/* /var/www/economikus/dist/
    sudo cp -r dist-server /var/www/economikus/
    sudo systemctl restart economikus
    ```

## 13. Проектирование интерфейса

В ходе проектирования интерфейса были подготовлены основные макеты страниц приложения, отражающие ключевые разделы и их структуру.

### 13.1. Структура публичной части

**Главная страница** построена по принципу последовательного раскрытия информации: hero-блок с заголовком и кнопками призыва к действию, блок преимуществ платформы (интерактивные курсы, практические навыки, сообщество, сертификаты), секция CTA с призывом к регистрации.

**Каталог курсов** содержит заголовок, панель фильтрации (поиск по названию, фильтр по уровню сложности, сортировка), сетку карточек курсов с пагинацией. Каждая карточка включает обложку, бейдж уровня, название, описание, теги, количество уроков и автора.

**Страница курса** включает hero-секцию с градиентным фоном, метаданными (уровень, количество уроков, длительность, студентов) и кнопкой «Начать обучение». Основной контент — аккордеон с модулями и уроками, боковая панель с информацией о курсе.

**Страница урока** содержит навигационную шапку с прогрессом, боковую панель со структурой курса, область с контентом (текст/видео/аудио/тест/калькулятор), кнопки навигации между уроками.

**Каталог калькуляторов** представляет сетку карточек калькуляторов с фильтрацией по категории. Каждый калькулятор — интерактивный компонент с полями ввода и результатами расчёта.

**Профиль пользователя** включает обложку, аватар, информацию о пользователе, вкладки: «О себе», «Мой прогресс», «История», «Избранное», «Сертификаты», «Подписки».

### 13.2. Панель автора

**Дашборд автора** содержит карточки статистики (просмотры, студенты, курсы, уроки), блоки быстрых действий («Мои курсы», «Мои уроки»), кнопку создания нового курса.

**Страница курсов автора** — таблица с курсами, их статусами, количеством уроков и просмотров, кнопками редактирования.

**Форма создания/редактирования курса** включает поля: название, slug, описание, обложка, уровень сложности, премиум-статус, теги, статус публикации.

**Страница управления модулями** — drag&drop-список модулей с уроками, возможность добавления, удаления и изменения порядка.

**Форма создания/редактирования урока** содержит поля: название, slug, описание, тип контента, модуль, длительность, премиум-статус, редактор контента в зависимости от типа (Markdown-редактор для текста, поле URL для видео/аудио, конструктор тестов для квизов).

### 13.3. Административная панель

**Дашборд администратора** включает карточки статистики платформы (пользователи, курсы, уроки, просмотры, премиум-пользователи), блоки быстрых действий («Новый курс», «Новый урок», «Новый тег»), статус системы.

**Страница управления пользователями** — таблица с пользователями, их ролями, статусом блокировки, датой регистрации, возможностью изменения роли и блокировки.

**Страница управления курсами** — таблица курсов с фильтрацией по статусу, возможностью модерации (публикация, архивирование), редактирования.

**Страница модерации контента** — список материалов на проверке, возможность одобрения или отклонения с комментарием.

**Страница заявок авторов** — таблица заявок на статус автора с возможностью одобрения или отклонения.

### 13.4. Цветовая гамма и типографика

Интерфейс выполнен в светлых тонах с акцентным бирюзовым цветом (#2A9D8F) для кнопок и активных элементов. Вспомогательные цвета: тёмно-синий (#264653) для текста и фонов, оранжевый (#F4A261) для акцентов, красный (#FF6B6B) для ошибок и удаления.

| Элемент | Цвет | HEX-код |

| Текст/Фон (вторичный) | Тёмно-синий | #264653 |

### 13.5. Адаптивность

Дизайн адаптируется под три основных breakpoints:

| Устройство | Ширина | Особенности |
|------------|--------|-------------|
| Десктоп | ≥1200px | Полный layout с боковыми панелями |
| Планшет | 768-1199px | Компактный layout, сворачиваемые панели |
| Мобильный | <768px | Одиночная колонка, выдвижное меню |
