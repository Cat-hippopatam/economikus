# План миграции Kakebo v1.0 → v2.0

> Стратегия перехода от базового учёта трат к полноценной системе финансового планирования

---

## 📌 Анализ текущего состояния (v1.0)

### ✅ Что работает стабильно

| Компонент | Статус | Примечание |
|-----------|--------|------------|
| Аутентификация | ✅ Стабильно | HttpOnly Cookie + `requireAuth` |
| CRUD записей | ✅ Стабильно | Все операции работают |
| Адаптивность | ✅ Стабильно | Mobile-first реализовано |
| Лимиты | ✅ Стабильно | С прогресс-баром и цветовой индикацией |

### ⚠️ Проблемы, которые нужно избежать в v2.0

1. **UUID вместо cuid** — все модели используют `@default(uuid()) @db.Char(36)`
2. **Порядок маршрутов** — конкретные маршруты (`/settings`) перед динамическими (`/:id`)
3. **Композитные unique keys** — использовать `{ profileId_year_month: { ... } }`
4. **Адаптивность форм** — поля в столбик на мобильных
5. **Типы Mantine** — проверять наличие props в текущей версии

---

## 🎯 Цели миграции

1. **Сохранить все существующие данные** — нулевая потеря записей
2. **Минимизировать downtime** — миграция должна быть однократной
3. **Обеспечить обратную совместимость** — временный fallback для старых API
4. **Избежать блокировок БД** — миграция на рабочем сервере без простоев

**❌ НИКОГДА НЕ ИСПОЛЬЗОВАТЬ:**
```bash
npx prisma db push --force-reset  # ❌ УДАЛИТ ВСЕ ДАННЫЕ!
npx prisma migrate reset           # ❌ УДАЛИТ ВСЕ ДАННЫЕ!
```

**✅ ИСПОЛЬЗОВАТЬ ТОЛЬКО:**
```bash
npx prisma db push                 # ✅ Применит изменения БЕЗ потери данных
npx prisma migrate dev             # ✅ Для dev-окружения (с миграциями)
npx prisma migrate deploy          # ✅ Для production (готовые миграции)
```

**Правило:**
- `--force-reset` и `reset` — **УДАЛЯЮТ** все таблицы и данные
- `db push` — **СОХРАНЯЕТ** все данные, только добавляет/изменяет структуру

#### 1.1 Добавить новые модели БЕЗ удаления старых полей
```prisma

model KakeboCategory {
  // ... (см. SCHEMA_V2.md)

  // ... (см. SCHEMA_V2.md)

  // ... (см. SCHEMA_V2.md)

  // ... (см. SCHEMA_V2.md)
}

// ✅ ДОБАВИТЬ новые поля в KakeboEntry
model KakeboEntry {
  id          String   @id @default(uuid()) @db.Char(36) // ← Изменить с cuid()
  // ... существующие поля
  
  categoryId  String?  // ← ДОБАВИТЬ (nullable для миграции)
  category    KakeboCategory? @relation(fields: [categoryId], references: [id])
  
  // ← ОСТАВИТЬ поле category enum временно (для fallback)
  categoryOld KakeboCategory? @map("category")
  
  isAutoGenerated Boolean @default(false) // ← ДОБАВИТЬ
}

// ✅ ОБНОВИТЬ KakeboSettings
model KakeboSettings {
  // ...
  income      Float?  // ← ДОБАВИТЬ (для бюджетного уравнения)
}
```

#### 1.2 Применить изменения БЕЗ потери данных
```bash
# ❌ НЕ ИСПОЛЬЗОВАТЬ: npx prisma db push --force-reset
#    (это УДАЛИТ ВСЕ данные в kakebo таблицах!)

# ✅ ИСПОЛЬЗОВАТЬ: npx prisma db push
#    (это применит изменения БЕЗ удаления данных)
npx prisma db push
```

**Важно:**
- `db push` применит изменения схемы к существующей БД
- Все данные в таблицах сохранятся
- Новые поля будут добавлены с значениями по умолчанию
- Старые поля останутся (для обратной совместимости)

#### 2.3 Скрипт миграции (Node.js)
```typescript
// scripts/migrate-kakebo-to-v2.ts

import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

async function migrate() {
  // 1. Создать системные категории (с русскими названиями)
  const categories = await Promise.all([
    prisma.kakeboCategory.upsert({
      data: { id: 'sys-life', name: 'Жизнь', type: 'SYSTEM', isEssential: true, order: 1 },
      where: { id: 'sys-life' },
      update: { name: 'Жизнь' }
    }),
    prisma.kakeboCategory.upsert({
      data: { id: 'sys-culture', name: 'Культура', type: 'SYSTEM', isEssential: true, order: 2 },
      where: { id: 'sys-culture' },
      update: { name: 'Культура' }
    }),
    prisma.kakeboCategory.upsert({
      data: { id: 'sys-extra', name: 'Дополнительное', type: 'SYSTEM', isEssential: false, order: 3 },
      where: { id: 'sys-extra' },
      update: { name: 'Дополнительное' }
    }),
    prisma.kakeboCategory.upsert({
      data: { id: 'sys-unexpected', name: 'Непредвиденное', type: 'SYSTEM', isEssential: false, order: 4 },
      where: { id: 'sys-unexpected' },
      update: { name: 'Непредвиденное' }
    })
  ])

  // 2. Перенести все записи
  const entries = await prisma.kakeboEntry.findMany()
  
  const categoryMap = {
    LIFE: categories[0].id,
    CULTURE: categories[1].id,
    EXTRA: categories[2].id,
    UNEXPECTED: categories[3].id
  }

  for (const entry of entries) {
    await prisma.kakeboEntry.update({
      where: { id: entry.id },
      data: {
        categoryId: categoryMap[entry.category]
      }
    })
  }

  console.log(`✅ Мигрировано ${entries.length} записей`)
}

migrate()
  .catch(console.error)
  .finally(() => prisma.$disconnect())
```
